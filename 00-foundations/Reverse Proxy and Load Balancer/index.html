Angular Nginx Reverse Proxy Load Balancer Enterprise Flow

1) Reverse Proxy — What it is

Definition: A reverse proxy sits in front of backend services and forwards requests to the correct service. The client talks to the reverse proxy (not directly to your backend servers).

Memory line: Reverse Proxy decides WHICH service.

Reverse Proxy Routing Diagram

Browser
   ↓
Reverse Proxy (Nginx)
   ↓           ↓
Angular App   Java Backend

Angular + API example (typical)

Nginx can serve Angular static files and proxy API requests at the same time:

server {

  listen 80;
  server_name library.com;

  # 1) Angular (static files)
  location / {
    root /var/www/angular;
    try_files $uri $uri/ /index.html;
  }

  # 2) APIs (proxy)
  location /api/ {
    proxy_pass http://java-backend:8080/;
  }

}

Why reverse proxy is used (in your Angular setup)

  • Routing: / goes to Angular; /api goes to backend services.
  • Hide backend: browser never sees internal backend hostnames/ports.
  • Central entry point: one domain (library.com) for everything.
  • Security controls: rate limits, headers, WAF-like rules (often).
  • SSL termination: reverse proxy can handle HTTPS certificates.

2) Load Balancer — What it is

Definition: A load balancer distributes traffic across multiple server instances to provide scalability and high availability.

Memory line: Load Balancer decides WHICH server.

Load Balancing Diagram

Browser
   ↓
Load Balancer
 ↓     ↓     ↓
S1    S2    S3

What load balancer solves

  • Scalability: handles more traffic by adding servers.
  • Availability: if one server fails, traffic goes to healthy ones.
  • Health checks: continuously checks which servers are alive.

3) Where each sits in the request-response flow

In a typical enterprise Angular production setup:

Browser
  ↓

  ↓

  ↓

  ↓

  ↓

Key idea: Load balancer spreads traffic across multiple Nginx (or app) servers; reverse proxy routes requests inside a chosen server to the right service.

5) Reverse Proxy vs Load Balancer — Differences

Topic Reverse Proxy Load Balancer
Main purpose Routing to the correct service (path/domain/headers) Distributing traffic across multiple servers
Decides… Which service handles the request Which server instance handles the request
Typical examples Nginx, Apache, Envoy AWS ELB/ALB, NLB, HAProxy, F5
Can one product do both? Yes. Many tools can act as both. The difference is the role in architecture, not always the product name.
Interview-ready line: Load balancer distributes traffic across servers for scalability/availability, while reverse proxy routes requests to appropriate backend services and hides internal topology from clients.
