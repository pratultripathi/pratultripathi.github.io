What is ACID?

ACID ┌────────────────────┐ │ A → Atomicity │ │ C → Consistency │ │ I → Isolation │ │ D → Durability │ └────────────────────┘
ACID is a set of guarantees provided by relational databases to ensure reliable and correct transactions.

A — Atomicity

Definition

Either the ENTIRE transaction succeeds or the ENTIRE transaction fails.
Transfer $100 from A → B Step 1: Deduct $100 from A ✅ Step 2: Add $100 to B ❌ Without Atomicity: Money disappears ❌
If any step fails: ROLLBACK everything ✅
Memory Trick → Atomicity = ALL or NOTHING

Interview Question

Q: What problem does Atomicity solve?

A: Atomicity prevents partial updates from being stored if part of the transaction fails.

C — Consistency

Definition

Database rules and constraints are ALWAYS preserved.
Rule: Balance cannot be negative Attempt: Balance = -500 Database → TRANSACTION REJECTED ❌
Memory Trick → Consistency = Rules always remain valid

I — Isolation

Definition

Concurrent transactions should not interfere with each other.
Only 1 ticket left User A buys ticket User B buys ticket simultaneously
Without Isolation: With Isolation:
Both succeed ❌ Oversold ticket ❌ One transaction succeeds ✅ Other waits/fails ✅

Interview Question

Q: Can you define different isolation levels ?

A:
Isolation Level Protection
Read Committed Prevents dirty reads
Repeatable Read Stable reads
Serializable Full protection
Memory Trick → Isolation prevents concurrency corruption

D — Durability

Definition

Once data is committed, it survives crashes.
User places order ✅ Database commits ✅ Power failure occurs ⚡ After restart: Order still exists ✅
Memory Trick → Durability = Saved forever

Interview Question

Q: How do databases achieve durability?

A: Using WAL logs, replication, checkpoints, and persistent disk storage.

ACID Summary

Property Meaning Memory Trick
Atomicity All or nothing No partial work
Consistency Rules preserved No invalid data
Isolation No interference Transactions stay separate
Durability Data survives crashes Saved forever
